Sidebar: HTTP/2

http://http2.github.io/faq/

New version of HTTP, mainly focused on improving page load performance

Based on SPDY from Google

  • Binary rather than textual
  • Multiplexed, not ordered/blocking
  • Single connection can load multiple resources in parallel
  • Header compression reduces overhead
  • Servers can proactively "push" data to client cache

Do You Use SSL?

SHA-1 Phase Out

Process has already started in Chrome and Firefox

Microsoft plans to withdraw support in Windows by 2017

  • SHA-1 first published in 1995
  • Viable attacks proven since 2005 (still theoretical)
  • SHA-2 uses similar algorithm, but much longer hashes
  • SHA-3 in the works

SHA-1 SSL certificates should be replaced ASAP to avoid this:

Project Spartan

Holy crap, a new browser! From Microsoft!

Announced Jan 21 – will ship with Windows 10

User features:

  • Web page annotation support
  • Cortana (voice command) integration

Project Spartan

  • New rendering engine (EdgeHTML.dll)
  • Ignores legacy document modes (X-UA-Compatible)
  • Loads IE 11 engine (Trident) for "enterprise legacy apps"
  • Both engines will ship in IE also
  • Self-updating ("evergreen")
  • Extensive ES 6 support, tons of new APIs
  • Too many features to list

More context from a member of the MS browser team

Project Spartan

Interesting context from a member of the MS browser team

On forking Trident:

It took almost 45 minutes just to process [the commit] (just committing the changes, not building!)...

In the coming months, swathes of IE legacy were deleted from the new engine. Gone were document modes. Removed was the subsystem responsible for emulating IE8 layout quirks. VBScript eliminated. Remnants like attachEvent, X-UA-Compatible, currentStyle were all purged from the new engine. The codebase looks little like Trident anymore (far more diverged already than even Blink is from WebKit).

Languages, Frameworks & Tools

React Native

Introduced at React.js Conf 2015

Deep dive keynote

  • Build native (iOS & Android) apps with JavaScript (or JSX) and React.js
  • Build with JS components, render to native UI views
  • Instant app refresh during dev (no compilation)
  • Compile-to-JS languages work too (CoffeScript, TypeScript)

Not much official info yet (reactnative.com)

Babel

http://babeljs.io/

Transpiles ES 6 code to ES 5 syntax (formerly "6to5")

Provides polyfills for many ES 6 features

Includes an interactive REPL

Aurelia

http://aurelia.io/

New "next generation" JS framework (introduction)

  • Written with ES 6/7
  • Integrates with web components (HTML compiler)
  • Small focused modules (monolithic or modular, your choice)
  • Convention-based MV* architecture
  • 2-way data binding, routing, dependency injection, etc.
