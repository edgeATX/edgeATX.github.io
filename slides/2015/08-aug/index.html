Trendspotting

Declarative Data Management

Today, developers mostly have to solve these themselves:

  • Local vs. remote data
  • Caching
  • Optimizing requests
  • Designing for graph retrieval

Relay

  • Released by Facebook on August 11
  • Works with FB's GraphQL (query language for graph data)
  • Components can describe the data necessary to render them
  • Automatic data consistency on client/server

Currently a tech preview, but used in production at FB

Falcor

No, not that one

Falcor

Created by Netflix

  • "One model everywhere" via a virtual JSON graph (sorta like GraphQL)
  • Object-oreiented data API via get/set, path syntax, routing
  • Transparent network calls, auto batching and de-duping
  • Side note: they are using this with React also!

Currently a developer preview, but used in production at Netflix

Summary

Specify what to fetch, but not necessarily how or where to get it

  • Declarative data definition
  • JSON-based
  • Designed for complex graph data
  • Transparent remote API
  • Request caching / batching / de-duping

Not quite ready for primetime, but watch this space...
