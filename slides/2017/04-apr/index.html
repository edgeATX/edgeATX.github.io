Security

Mandatory CAA Checking

What's a CAA?

  • "Certification Authority Authorization"
  • A new-ish DNS resource record type
  • Heard of A, CNAME and TXT records? Now there's CAA too

Mandatory CAA Checking

The purpose is basically 2 things:

  • Restrict which CA's can issue certificates for your domain
  • Define who gets notified if this is violated

Find out more. Here's an example:

example.com.  CAA 0 issue "letsencrypt.org"
example.com.  CAA 0 iodef "mailto:example@example.com"

Mandatory CAA Checking

Cool! Why should I care?

  • Helps prevent hackers from exploiting your site
  • It's free and trivial to set up
  • In March the CA/Browser Forum made checking mandatory for CA's
Bottom line: If you manage DNS, start using this!

In Case You
Missed It

Avoid JS Dialogs

The Chromium team highly recommends that you not use JavaScript dialogs.

Includes alert(), confirm(), and prompt()

They recommend instead using...

  • HTML5 <dialog> (or similar) for user input
  • Notifications API for alerts
  • Page Visibility API insted of unbeforeunload
